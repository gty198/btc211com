📌 什么是 Copilot AI 代理

GitHub Copilot 编程代理（Coding Agent）是 GitHub Copilot 的一项功能，它可以在后台自主完成编程任务。你可以通过 GitHub Issue 或 PR 直接指派任务给 Copilot，它会自动创建分支、编写代码、运行测试，并提交 Pull Request 供你审查。

此外，你还可以在 VS Code 中使用 Copilot Agent Mode（代理模式）进行本地交互式开发，让 AI 帮你完成复杂的多步骤编码任务。

✅ 前提条件

  • 一台运行 macOS 的 Mac 电脑
  • 一个 GitHub 账号
  • 订阅了 GitHub Copilot（个人版、商业版或企业版均可）
  • 安装了 Visual Studio Code（VS Code）

🖥️ 方式一：在 VS Code 中使用 Agent Mode

Agent Mode 是 VS Code 中的 Copilot 交互模式，可以自主执行多步骤任务，包括编辑文件、运行终端命令等。

1安装 VS Code

从官网下载并安装 macOS 版本：

https://code.visualstudio.com/

2安装 GitHub Copilot 扩展

打开 VS Code，点击左侧扩展图标（或按 ⇧⌘X），搜索并安装：

  • GitHub Copilot
  • GitHub Copilot Chat

安装完成后，点击右下角弹出的提示登录你的 GitHub 账号。

3打开 Copilot Chat 面板

使用快捷键 ⌃⌘I（Control + Command + I）打开 Copilot Chat 面板。

4切换到 Agent Mode

在 Chat 面板顶部的下拉菜单中，从"Ask"或"Edit"切换到 "Agent" 模式。

5开始使用

在 Agent 模式下，你可以用自然语言描述任务，例如：

帮我创建一个 React 登录页面，包含用户名和密码输入框

Copilot 会自动分析项目结构、创建/修改文件、运行终端命令来完成任务。每一步操作都会征求你的确认。

💡 提示：Agent Mode 可以执行终端命令（如安装依赖、运行测试），每次执行前都会请求你的许可。

☁️ 方式二：使用 GitHub 编程代理（Coding Agent）

编程代理在 GitHub 的云端环境中运行，无需本地配置开发环境。你只需通过 GitHub 网站或 GitHub Mobile 来指派任务。

1确认权限

确保你的仓库或组织已启用 Copilot 编程代理功能。在仓库的 Settings → Copilot → Coding agent 中检查。

2通过 Issue 指派任务

在 GitHub 仓库中创建一个 Issue，描述你想要完成的任务。然后将该 Issue 指派给 Copilot（在 Assignees 中选择）。

Copilot 会自动：

  1. 分析 Issue 内容
  2. 创建一个新分支
  3. 编写代码并提交
  4. 创建 Pull Request
3通过 PR 评论指派任务

你也可以在已有的 Pull Request 中 @copilot 来让 Copilot 完成代码审查反馈或修改。

4使用 GitHub Mobile

在 iPhone 或 iPad 上安装 GitHub Mobile，可以随时随地向 Copilot 指派编程任务。

💡 提示：编程代理会在安全的云端沙箱中运行你的代码，所有更改都通过 Pull Request 提交，你可以在合并前完整审查。

⌨️ Mac 上的常用 Copilot 快捷键

  • ⌃⌘I — 打开 Copilot Chat 面板
  • ⌘I — 在编辑器中打开内联 Chat
  • Tab — 接受 Copilot 的代码建议
  • Esc — 拒绝 Copilot 的代码建议
  • ⌥] — 查看下一个建议
  • ⌥[ — 查看上一个建议

❓ 常见问题

Q: 为什么看不到 Agent Mode 选项？

请确保 VS Code 和 Copilot 扩展已更新到最新版本。Agent Mode 需要较新版本的 VS Code 才能使用。

Q: 编程代理不响应我的 Issue？

请检查：

  • 你的 Copilot 订阅是否有效
  • 仓库设置中是否已启用 Coding agent
  • Issue 是否已正确指派给 Copilot

Q: Agent Mode 和编程代理有什么区别？

Agent Mode 在本地 VS Code 中运行，适合交互式开发；编程代理在 GitHub 云端运行，适合异步任务处理，无需保持电脑在线。
